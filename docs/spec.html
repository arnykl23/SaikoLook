1. 目的とスコープ

最終ゴール: メール見落とし防止アプリ。Outlook / Gmail を統合し、LLM が各メールの 「重要度」「タスクの重さ」を推定してダッシュボードに統合表示する Web サイト。

本 PoC のスコープ: 第一歩として「Gmail から受信メールを取得し、共通スキーマに正規化して API で返す」までを実装・検証する。

・Gmail 受信トレイから最新メール取得
・件名/差出人/宛先/日時/未読/スニペットの正規化
・GET /emails で JSON 配列を返す API
・CLI 表示（動作確認用）

・Web ダッシュボード UI

・マルチユーザ

4. app/models.py — 共通スキーマ（全解説）

app/models.py
from datetime import datetime
from pydantic import BaseModel


class EmailMessage(BaseModel):
    id: str                       # 取得元の安定識別子（Gmail は IMAP UID）
    provider: str = "gmail"       # "gmail" | "outlook" など
    subject: str = ""
    sender: str = ""              # "Name <addr>"
    to: list[str] = []
    received_at: datetime | None = None
    snippet: str = ""             # 本文先頭のプレビュー
    is_unread: bool = False

    # 次フェーズ(LLM)で埋める想定の余地（PoC では未使用）
    # body_text: str | None = None
    # importance: int | None = None
  • BaseModel を継承 — pydantic のモデル。型に基づき自動でバリデーションと JSON シリアライズを行う。FastAPI の response_model にそのまま使える。
  • id: str — メールの安定識別子。Gmail では IMAP UID を入れる（後述）。位置で変わるシーケンス番号は使わない。
  • provider — 取得元の種別。将来 Outlook を足したとき "outlook" を入れて区別する。デフォルト "gmail"。
  • subject/sender — 件名・差出人。デコード済みの可読文字列を入れる。
  • to: list[str] — 宛先アドレスのリスト。
  • received_at: datetime | None — 受信日時。パース失敗時は None。FastAPI が ISO 文字列に自動変換。
  • snippet — 本文プレビュー（先頭120字）。
  • is_unread — 未読フラグ。
  • コメントアウトの2行 — 次フェーズで本文全文 (body_text)・LLM スコア (importance) を足す予定地。スキーマ拡張で UI/API はそのまま使える設計意図を明示。

5. app/fetch_gmail_imap.py — 取得層（全解説）

このファイルが心臓部。役割ごとに関数を分け、fetch_emails() が中核。

5.1 import と定数

app/fetch_gmail_imap.py — 冒頭
import email
import email.message
import imaplib
import os
import re
from email.header import decode_header, make_header
from email.utils import parsedate_to_datetime

from app.models import EmailMessage

IMAP_HOST = "imap.gmail.com"
IMAP_PORT = 993
IMAP_TIMEOUT = 10  # 秒。接続/ログイン/取得が詰まってワーカーが固まるのを防ぐ
MAX_RESULTS = 10
PREVIEW_BYTES = 16384
  • imaplib — Python 標準の IMAP クライアント。これだけで Gmail に接続できるので追加依存が要らない。
  • email 系 — 取得した生メールのパースとデコードに使う標準ライブラリ。decode_header/make_header は =?UTF-8?B?...?= 形式のヘッダ復元、parsedate_to_datetime は日付文字列→datetime。
  • re — FETCH 応答テキストから UID を正規表現で抜くのに使用。
  • IMAP_TIMEOUT=10 — ソケットタイムアウト（秒）。接続/ログイン/取得が固まってもワーカーが無限に詰まらないようにする安全弁。
  • MAX_RESULTS=10 — 取得件数の既定。
  • PREVIEW_BYTES=16384 — 1通あたりの取得上限バイト。ヘッダはメール先頭にあるので16KBで件名等＋本文先頭が取れ、巨大な添付込みメールでも全文を落とさない。

5.2 decode() — ヘッダの文字化け対策

app/fetch_gmail_imap.py
def decode(value: str | None) -> str:
    if not value:
        return ""
    try:
        return str(make_header(decode_header(value)))
    except Exception:
        return value
  • 件名や差出人は =?UTF-8?B?...?= のようにエンコードされていることが多い（日本語メールで頻出）。
  • decode_header() でパーツ分解 → make_header() で結合 → str() で可読文字列に復元。これが無いと件名が文字化けする。
  • None/空は ""、復元失敗時は元値をそのまま返す防御的実装。

5.3 extract_snippet() — 本文プレビュー抽出

app/fetch_gmail_imap.py
def extract_snippet(msg: email.message.Message, limit: int = 120) -> str:
    """text/plain を優先して本文の先頭を取り出す。"""
    text = ""
    if msg.is_multipart():
        for part in msg.walk():
            if part.get_content_type() == "text/plain" and not part.get_filename():
                payload = part.get_payload(decode=True) or b""
                charset = part.get_content_charset() or "utf-8"
                text = payload.decode(charset, errors="replace")
                break
    else:
        payload = msg.get_payload(decode=True) or b""
        charset = msg.get_content_charset() or "utf-8"
        text = payload.decode(charset, errors="replace")

    text = " ".join(text.split())  # 改行・連続空白を畳む
    return text[:limit]
  • メールは「テキスト版＋HTML版＋添付」が同梱(multipart)なことが多い。
  • msg.walk() で全パートを巡回し、text/plain かつ添付でない(get_filename() が無い)パートを採用。
  • get_payload(decode=True) で base64/quoted-printable を自動デコードしてバイト列を取得 → そのパートの charset で文字列化（不明なら UTF-8、壊れた文字は errors="replace" で置換）。
  • 非 multipart のメールは本文がそのまま payload なので、その分岐で処理。
  • " ".join(text.split()) で改行・連続空白を1つに畳み、[:limit] で先頭120字に切り詰め。

5.4 _get_credentials() — 認証情報の取得

app/fetch_gmail_imap.py
def _get_credentials() -> tuple[str, str]:
    address = os.environ.get("GMAIL_ADDRESS", "").strip()
    app_password = os.environ.get("GMAIL_APP_PASSWORD", "").replace(" ", "")
    if not address or not app_password:
        raise RuntimeError(
            "環境変数 GMAIL_ADDRESS / GMAIL_APP_PASSWORD が未設定です "
            "(secrets/gmail.env を確認してください)。"
        )
    return address, app_password
  • 認証情報はコードに書かず環境変数から読む（secrets/gmail.env 由来）。
  • アプリパスワードは表示が llcb kglc ... のように空白区切りなので .replace(" ", "") で詰める。env にどちらの形式で書いても通る。
  • 未設定なら RuntimeError。これは上位（API/CLI）で適切に変換される（後述）。

5.5 _connect() — IMAP 接続

app/fetch_gmail_imap.py
def _connect() -> imaplib.IMAP4_SSL:
    address, app_password = _get_credentials()
    # timeout を付け、接続/ログインが固まってもワーカーが無限に詰まらないように。
    imap = imaplib.IMAP4_SSL(IMAP_HOST, IMAP_PORT, timeout=IMAP_TIMEOUT)
    try:
        imap.login(address, app_password)
    except imaplib.IMAP4.error as e:
        raise RuntimeError(
            f"ログイン失敗: {e} / 2段階認証とアプリパスワードを確認してください。"
        )
    return imap
  • IMAP4_SSL(... , timeout=10) — imap.gmail.com:993 に SSL 接続。タイムアウト付きで可用性リスクを低減（レビュー対応）。
  • ログイン失敗は RuntimeError に変換し、原因（2段階認証/アプリパスワード）を案内。
  • INBOX の選択はここでは行わず、件数取得とまとめて fetch_emails() 側で行う。

5.6 _build_message() — 1通を EmailMessage に変換

app/fetch_gmail_imap.py
def _build_message(descriptor: bytes, raw: bytes) -> EmailMessage:
    # FETCH 応答の記述子から UID と既読状態を取り出す。
    uid_match = re.search(rb"UID (\d+)", descriptor)
    uid = uid_match.group(1).decode() if uid_match else ""
    is_unread = b"\\Seen" not in descriptor

    msg = email.message_from_bytes(raw)
    try:
        received = parsedate_to_datetime(msg.get("Date"))
    except Exception:
        received = None

    to = [a.strip() for a in decode(msg.get("To")).split(",") if a.strip()]
    return EmailMessage(
        id=uid,
        provider="gmail",
        subject=decode(msg.get("Subject")),
        sender=decode(msg.get("From")),
        to=to,
        received_at=received,
        snippet=extract_snippet(msg),
        is_unread=is_unread,
    )
  • 引数は2つ。descriptor＝FETCH 応答のメタ行（例 b"3 (UID 4 FLAGS (\\Seen) BODY[]<0> {16384}"）、raw＝メール本体バイト列。
  • UID 抽出 — re.search(rb"UID (\d+)", ...) で記述子から UID を取得。これを id にすることで安定識別子になる（シーケンス番号を使わない＝レビュー対応）。
  • 未読判定 — 記述子に \Seen フラグが無ければ未読。
  • email.message_from_bytes(raw) — 生バイトを Message にパース。以降 msg.get("Subject") 等でヘッダ取得。
  • 受信日時 — Date ヘッダを datetime 化。失敗時 None。
  • 宛先 — To をデコードしてカンマ分割（簡易）。
  • 最後に各フィールドを詰めて EmailMessage を生成して返す。

5.7 fetch_emails() — 取得の中核

app/fetch_gmail_imap.py
def fetch_emails(limit: int = MAX_RESULTS) -> list[EmailMessage]:
    """受信トレイの最新メールを共通スキーマで返す（新しい順）。"""
    imap = _connect()
    try:
        # SELECT の応答に総件数(EXISTS)が入る。これを使い、全件 SEARCH せず
        # 末尾(=最新) limit 件だけを位置指定で取得する（大きい受信箱でも軽量）。
        status, data = imap.select("INBOX", readonly=True)
        if status != "OK":
            raise RuntimeError(f"INBOX を開けませんでした: {data}")
        total = int(data[0]) if data and data[0] else 0
        if total == 0:
            return []

        start = max(1, total - limit + 1)
        seq_set = f"{start}:{total}"  # 末尾 limit 件のシーケンス範囲（このセッション内限定）
        # UID も同時取得して安定IDにする。先頭 PREVIEW_BYTES のみ・PEEK で既読化しない。
        status, msg_data = imap.fetch(
            seq_set, f"(UID FLAGS BODY.PEEK[]<0.{PREVIEW_BYTES}>)"
        )
        if status != "OK":
            raise RuntimeError("メールの取得に失敗しました。")

        results = [
            _build_message(item[0] or b"", item[1])
            for item in msg_data
            if isinstance(item, tuple)
        ]
        results.reverse()  # シーケンス昇順 → 新しい順に
        return results
    finally:
        try:
            imap.logout()
        except Exception:
            pass
  • imap.select("INBOX", readonly=True) — 受信トレイを読み取り専用で開く（既読化しない）。応答 data[0] に総件数(EXISTS)が入る。
  • 失敗チェック — SELECT が OK でなければ RuntimeError（レビュー指摘の「失敗をエラー扱い」対応）。
  • total と件数0処理 — 件数を int 化。0 件なら空リストを返す。
  • 全件 SEARCH をしない — start = total - limit + 1 として末尾 limit 件のシーケンス範囲 start:total だけを対象にする。大きい受信箱でも全 UID 列挙のコストを払わない（レビュー対応の肝）。シーケンス番号はこの選択セッション内だけの一時的な位置指定に使い、外部に返さない。
  • imap.fetch(seq_set, "(UID FLAGS BODY.PEEK[]<0.16384>)") — 範囲内の各メールについて、UID・フラグ・先頭16KB本文を一括取得。PEEK なので既読フラグを変えない。
  • 結果組み立て — 応答リストの各タプルを _build_message() で EmailMessage 化。tuple でない区切り要素は除外。
  • results.reverse() — シーケンス昇順（古い順）で返るので反転し新しい順に。
  • finally: imap.logout() — 例外の有無に関わらず必ず接続を閉じる。

5.8 main() — CLI 表示

app/fetch_gmail_imap.py — 末尾
def main() -> None:
    try:
        emails = fetch_emails()
    except RuntimeError as e:
        raise SystemExit(str(e))

    if not emails:
        print("受信トレイにメールが見つかりませんでした。")
        return

    print(f"\n===== 受信トレイ 最新 {len(emails)} 件 =====\n")
    for i, m in enumerate(emails, 1):
        date = m.received_at.isoformat() if m.received_at else ""
        print(f"[{i}] {'●未読' if m.is_unread else '  既読'}  {date}")
        print(f"    From   : {m.sender}")
        print(f"    Subject: {m.subject}")
        print(f"    Snippet: {m.snippet}")
        print()


if __name__ == "__main__":
    main()
  • 同じ fetch_emails() を使い、結果をターミナルに整形表示する CLI。
  • エラー変換 — RuntimeError を SystemExit にして、メッセージ表示＋終了コードで終わる（CLI らしい振る舞い）。API 側は同じ例外を HTTP 500 に変換する＝層ごとに適切なエラー処理。
  • ●未読/既読 を付けて一覧表示。
  • if __name__ == "__main__" — python -m app.fetch_gmail_imap で実行したときだけ main() が動く。import 時には動かないのでライブラリとしても安全。

6. app/main.py — API層（全解説）

app/main.py
import os

from fastapi import FastAPI, HTTPException, Query
from fastapi.middleware.cors import CORSMiddleware

from app.fetch_gmail_imap import MAX_RESULTS, fetch_emails
from app.models import EmailMessage

app = FastAPI(title="ReplyGuard API", version="0.1.0")

# CORS は許可 origin を明示（ワイルドカードは使わない）。
_default_origins = (
    "http://localhost:3000,http://127.0.0.1:3000,"
    "http://localhost:5173,http://127.0.0.1:5173"
)
ALLOWED_ORIGINS = [
    o.strip()
    for o in os.environ.get("ALLOWED_ORIGINS", _default_origins).split(",")
    if o.strip()
]

app.add_middleware(
    CORSMiddleware,
    allow_origins=ALLOWED_ORIGINS,
    allow_methods=["GET"],
    allow_headers=["*"],
)


@app.get("/health")
def health() -> dict:
    return {"status": "ok"}


@app.get("/emails", response_model=list[EmailMessage])
def get_emails(limit: int = Query(MAX_RESULTS, ge=1, le=50)) -> list[EmailMessage]:
    try:
        return fetch_emails(limit=limit)
    except RuntimeError as e:
        raise HTTPException(status_code=500, detail=str(e))
  • app = FastAPI(...) — アプリ本体。title/version は Swagger に表示される。
  • CORS 設定 — ALLOWED_ORIGINS を環境変数（既定はローカル開発ポート）から構築し、ワイルドカードを使わず明示した origin だけ許可。これにより別オリジンの悪意あるサイトがブラウザ経由で /emails のレスポンスを読むのを防ぐ（レビュー対応）。
  • allow_methods=["GET"] — 読み取り専用 API なので GET のみ許可。
  • /health — 死活監視・起動待ち判定用。{"status":"ok"} を返す。
  • /emails — メイン。response_model=list[EmailMessage] で出力型を保証し、自動 JSON 化＋Swagger ドキュメント生成。
  • limit: int = Query(10, ge=1, le=50) — クエリパラメータ。1〜50 の範囲バリデーション付き。範囲外は FastAPI が 422 で弾く。
  • エラー変換 — 取得層の RuntimeError（認証失敗・取得失敗）を HTTP 500 に変換して返す。
  • この層は薄いのが要点。実処理は持たず取得層に委譲する。

補足: app/__init__.py は空ファイルだが、app/ をパッケージ化し from app.xxx import ... や uvicorn app.main:app、python -m app.fetch_gmail_imap を成立させるために必要。

  • api — uvicorn で FastAPI を起動。--host 0.0.0.0 はコンテナ内で待受、ホスト公開は 127.0.0.1:8000 に限定（LAN へ出さない）。
  • gmail-poc — CLI 一覧表示。tty 有効で整形出力を見やすく。
  • gmail-poc-oauth — OAuth 版の予備。secrets/ をマウントし credentials.json を渡す想定。現状は未使用。

10. セキュリティ / レビュー対応

Codex アドバサリアルレビュー（実装方針への挑戦的レビュー）の指摘と対応履歴。

指摘重大度対応
IMAP が RFC822 でメール全文（添付込み）取得 → 巨大メールで失敗・遅延medium 修正済 BODY.PEEK[]<0.16384> で先頭16KBに限定。
ワイルドカード CORS で受信トレイが他オリジンから読めるhigh 修正済 allow_origins を明示・GET 限定。
ID に不安定な IMAP シーケンス番号を使用medium 修正済 UID を取得・採用。位置指定は同一セッション内限定。
タイムアウト無し＋毎回 UID SEARCH ALL で全件列挙 → ワーカー詰まり/大受信箱で重いmedium 修正済 timeout=10 追加。SELECT の件数から末尾N件だけ範囲取得し全件 SEARCH を廃止。SELECT/FETCH 失敗をエラー化。
/emails に認証境界が無い（CORS は認証ではない）high 見送り PoC のローカル用途のため未対応。本格化/共有時に env トークン照合を追加予定。
OAuth コールバックを 0.0.0.0 公開（予備の OAuth 版）high 対象外 OAuth 版は未使用。ホスト公開は 127.0.0.1 限定に変更済み。

その他の運用上の配慮

  • 認証情報は secrets/ に隔離し .gitignore で除外。
  • IMAP は読み取り専用（readonly=True）＋ PEEK で既読フラグを変更しない。
  • スコープは受信メールの読み取りのみ。送信権限は持たない。
  • ホストへの公開は全サービス 127.0.0.1 限定（LAN 非公開）。

11. 制約と次の一手

既知の制約

  • スニペットは text/plain 優先。HTML のみのメールはテキスト抽出が簡易。
  • INBOX 限定・最大50件。ページング/検索条件/フォルダ指定は未実装。
  • 本文は先頭16KBのみ（全文・添付は次フェーズ）。
  • /emails は未認証（ローカル PoC 前提）。
